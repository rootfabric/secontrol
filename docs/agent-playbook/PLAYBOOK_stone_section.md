### Добыча камня (Stone) через Nanobot Drill

Stone — это особый сценарий. Не обрабатывай Stone как обычную руду.

Для обычной руды агент ищет залежь в SharedMap или через ore scanner.
Для Stone этого делать не нужно:

- не ищи `Stone deposit` в SharedMap;
- не требуй GPS рудной жилы;
- не требуй астероид с конкретным материалом;
- не останавливай миссию из-за отсутствия `Stone` в SharedMap;
- не делай вывод “камень не найден”, если есть любой доступный воксель.

Правило:

> Чтобы добывать Stone, нужно навести область Nanobot Drill на любой блок вокселя/астероида/породы и включить фильтр добычи Stone.

Минимальная логика агента:

1. Найти грид, указанный пользователем.
2. Проверить, есть ли на гриде Nanobot Drill.
3. Найти ближайший доступный воксель:
   - астероид рядом;
   - поверхность планеты;
   - уже видимый voxel/radar contact;
   - любой voxel block в зоне действия Nanobot.
4. Навести Nanobot area на этот воксель.
5. Включить Collect mode.
6. В фильтре Collect выбрать именно Stone.
7. Запустить Nanobot Drill.
8. Проверять рост Stone в инвентарях грида.
9. Остановить добычу после достижения нужного количества или заполнения доступного инвентаря.

Важно:

- Cargo Container не является обязательным hard-block. Используй любой доступный инвентарь грида: Nanobot, cockpit, connector, cargo, refinery, assembler и т.п.
- Remote Control, Beacon и Connector не обязательны для стационарной добычи Stone.
- Если грид — ровер, не пытайся использовать космическую навигацию как обязательное условие.
- Если ровер уже стоит рядом с вокселем, можно добывать на месте.
- Если Stone нужен “100 т”, считай это как `100000 kg`.

Запрещено:

- искать Stone через `shared_map_deposits.py --material Stone` как обязательный шаг;
- прекращать миссию только потому, что нет RC/Beacon/Connector;
- писать “негде хранить камень” только из-за отсутствия Cargo Container;
- запускать обычную рудную миссию Uranium/Platinum для Stone;
- считать Stone загрязнением при задаче добыть Stone.
