# Grind Color Investigation — хронология

## Проблема (2026-05-16)

Наносборщик не разбирал солнечную панель, хотя железный блок с тем же цветом разбирался.

## Исследование

### Этап 1: неправильный grind color

Отправили `set_grind_color([321, 100, 50])` через API — мод проигнорировал.
Покрасили панель в HSV (321, 100, 50) → ColorMaskHSV (0.8917, 1.0, 0.0).
Железный блок имел ColorMaskHSV (0.8917, 0.2, 0.05) — **другой цвет!**

### Этап 2: неверная формула конвертации

Формула `S_game/100` дала S=1.0 вместо S=0.6. Оказалось что формат терминала
игры НЕ совпадает напрямую с ColorMaskHSV.

### Этап 3: правильная формула

```python
api_S = (colorMask_y + 1) / 2  # y=0.2 → 0.6
api_V = (colorMask_z + 1) / 2  # z=0.05 → 0.525
```

Покрасили панель: `paint_block(hsv=[321, 0.6, 0.525])` → ColorMaskHSV (0.8917, 0.200000048, 0.0499999523).

### Этап 4: подозрение на тип блока

При идентичном ColorMaskHSV железный блок разбирался, а панель — нет.
Предположили что наносборщик не разбирает functional блоки.

### Этап 5: разборка после включения

После повторного включения наносборщика (`set_enabled(True)`) оба блока
были разобраны за 60 секунд. Проблема была в том, что наносборщик не был
включён или grind mode не был активен в предыдущих попытках.

## Выводы

1. **Nanobot разбирает ВСЕ типы блоков** (structural + functional) при точном совпадении цвета
2. **Формула конвертации**: api_S = (y+1)/2, api_V = (z+1)/2 (из ColorMaskHSV)
3. **set_grind_color API не работает** — настройка только вручную в игре
4. **Telemetry не содержит grind state** — нельзя проверить через API
5. **Device IDs меняются** между сессиями — всегда обнаруживать заново
